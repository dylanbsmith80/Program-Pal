# Exercise Entry Schema

Use this schema when Program Pal needs to describe, evaluate, or store an exercise choice.

## Exercise Entry

### Exercise Name

The common name of the exercise.

### Source Status

State one:

- `Verified`: Found in ACE, NASM, NSCA, the trainer's approved notes, or another approved Program Pal reference.
- `Trainer Provided`: Directly provided by the trainer.
- `Unverified`: From a non-verified source or general model knowledge.

If `Unverified`, apply extra caution and prefer a verified alternative when possible.

### Source Link Or Note

Include the source link or note when useful.

### Placement

State one:

- Warm Up
- Workout
- Cooldown

### Category

Examples:

- Strength
- Hypertrophy
- Mobility
- Flexibility
- Conditioning
- Core
- Corrective
- Balance
- Power
- Skill
- Recovery

### Movement Pattern

Examples:

- Squat
- Hinge
- Lunge
- Push
- Pull
- Carry
- Rotation
- Anti-rotation
- Anti-extension
- Gait
- Jump
- Crawl
- Single-leg
- Shoulder mobility
- Hip mobility
- Thoracic mobility

### Primary Purpose

Explain what the exercise is meant to accomplish in this program.

### Best For

List client goals, qualities, or contexts where the exercise fits well.

### Use Caution When

List pain, injuries, constraints, equipment issues, or skill limitations that should make Program Pal cautious.

### Equipment Needed

List required equipment.

### Difficulty

State one:

- Beginner
- Intermediate
- Advanced

### Coaching Notes

Short cues or setup notes a trainer can use.

### Programming Notes

Sets, reps, duration, rest, tempo, intensity, placement, or progression notes when relevant.

### Regression

A simpler or safer option that preserves the training purpose.

### Progression

A harder option or next step that preserves the training purpose.

### Substitutions

Other exercises that can serve a similar role.

### Client-Specific Rationale

Explain why this exercise fits the specific client, not just why the exercise is generally useful.
